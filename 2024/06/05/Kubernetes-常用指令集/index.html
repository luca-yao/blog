Kubernetes 常用指令集

2024-06-05
Services
346 詞

    前言

    以下不是甚麼專業的經驗分享，只是把常用的指令集做一個歸類與說明，方便日後查找，當然你也可以去查官方原文件啦，但感覺不夠直覺，所以就放這吧(＊゜ー゜)b

    基礎指令

    命令 说明
    kubectl create 利用 yaml/json 文件或者標準輸入建立一個資源 例如：namespace/pod/deployment/service等
    kubectl expose 將 yaml/json 文件中定義的資源對象的Port 暴露给新的 service
    kubectl run 建立並運行一個或多個image
    kubectl set 配置资源对象设置特定功能
    kubectl explain 用來查詢 Kubernetes 中各種資源類型的配置參數內容，包括 Pod、Service、Deployment 等。
    kubectl get 獲取一個或多個資源的資訊
    kubectl edit 使用默認編輯器編輯主機上定義的服務，編輯完後會自動套用
    kubectl delete 刪除 Kubernetes 集群中的資源。這個命令可以用來刪除各種類型的資源，例如 Pod、Service、Deployment 等

    部署命令 DeployCommands

    命令 说明
    kubectl rollout 资源管理对象的部署
    kubectl rollout-update 使用 rc（replication controller）来做滚动更新
    kubectl scale 扩容或者缩容 deployment replicaset replication contrller 等
    kubectl autoscale 自动设置在 k8s 系统中运行的 pod 数量（水平自动伸缩）
